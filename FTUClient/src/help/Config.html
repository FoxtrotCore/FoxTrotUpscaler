Config.txt Usage

Overview

When you first run FoxTrotUpscaler, it will generate a file called config.txt in the same folder the Jar file is placed in. It has some extra text for the user to read, but at its core, its a list of key/value pairs that the User MUST edit if they want a good upscale. The default values are supplied so that the program will work, and some you may leave alone. However, they are all worth looking into, as all the options the dependencies provide that do not fit the purpose of FTU are hard-coded anyway. They are listed at the bottom of the config.txt in case the user wishes to know exactly. If you screw up the syntax, the program will delete the old one and create a fresh one with the default values, so don’t screw it up -Eventually rename instead of delete

Exceptions

Almost all of the options require a value, except for

  1. Stage A Input Source Arguments (line 20)
  2. Stage A Export Arguments (Line 21)
  3. Stage D Input Source Arguments (line 22)

Options

The entire list of values required can be organized into 3 parts.

  1. Waifu2x-caffe Upscale Parameters
  2. Video Frame rates
  3. FFMPEG Arguments

Waifu2x-caffe Upscale Parameters

The First set of values are all required for Waifu2x (Stage B) to function. The values define how Waifu2x upscales the content, as well as the resources available to it. You can find an explanation for all of these parameters on the developers website: https://github.com/lltcggie/waifu2x-caffe
(Yes, it is in Japanese, but it is understandable with Google Translate)

The First half of the GitHub page should explain what the options do. Some affect how the upscale is done, and affect the quality. Other options affect how fast the upscale happens, which is dependent on your hardware. The Second half will explain the syntax for the CUI variant of the software. The command examples shown are intended to be entered like “-tta 0” into CMD, but that is FTU’s job, you just have to enter “0” in line 15 after the “>” character, to enter the equivalent value.

Video Frame rates

The Next values are the Import and Export videos frame rate. Just find the first in VLC, pick the second and enter them in. Make sure to double check this value every time you change “TV show” or whatever. The reason these values are in the text file rather then the GUI is that the values are expected to be consistent for a "group" of videos. So rather then have you enter it in 100 times, just change it in the text file when needed.
It is OK, to use multiple versions of a config, in the same queue of processes. You do not even have to close FTU to make a change in this file.

FFMPEG Arguments

The Last set of values are the FFMPEG arguments. These are a lot less like multiple choice questions, and more like free response. They can be easier to mess up if you are unfamiliar with FFMPEG. So I suggest you brush up on the documentation, because this can allow you to use the program more effectively. https://ffmpeg.org/documentation.html

The 6 Arguments all work in the exact same way, but apply to different parts of all the Stages that use FFMPEG. Each of them provide extra/required FFMPEG arguments of your choosing to the mentioned content that it refers to.
For example, one of the required arguments for “Stage D Export” provide a video encoder and preset to FFMPEG,. In Stage D, the program is converting individual frames into a video, which is why it requires an encoder, as it cant just copy the stream.

However, you could also append to any of the 6 arguments, other flags that FFMPEG would recognize. This can allow you to edit the final product an infinite number of ways, without needing a second pass though FFMPEG or re-encoding. Otherwise, you can put the bare minimum of the required flags, and for the optional arguments (the 3 Exceptions above) , leave them blank.

Just beware that FTU will not confirm any of the content in the Arguments, and assume you at least put the required parameters. If it crashes due to User error in the config.txt, FFMPEG should tell you what it did not like in the Console Area, so you can fix it. Any further help with this should be directed to FFMPEG’s documentation.
