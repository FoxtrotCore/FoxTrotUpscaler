File System

Overview

Starting FTU for the first time, will generate a bunch of Folders and a few files. All of the work done by FTU will be done in these folders, so you should become knowledgeable about them. Some will be more useful then others. This section will explain each of them, as well as detail the expected file names, for files in the said folders.
Note, if a <> has number signs in them, it signified how many chars/digits are required for that section, no more, no less.
All LANGs refer to a standarized 3 digit ISO 639-2/B language code, see here for the entire list: https://en.wikipedia.org/wiki/List_of_ISO_639-1_codes

Commonly Accessed Folders

/Export

This folder is where all of the completed upscaled videos will go.This is also where Videos still in the process of Stage D, or videos that were aborted in Stage D are.This is where Re-muxed Videos will go on completion.

  • Completed Upscales: "Upscaled_<LANG ###>_<EP ###>_<SHOWNAME>.<EXTENSION>"
  • Completed Re-muxes: "Remuxed_<LANG ###>_<EP ###>_<SHOWNAME>.<EXTENSION>"
  • Incomplete/Aborted Videos: "<EP ###>_<SHOWNAME>.<EXTENSION>"

/ImportVideo

This folder is where you put all of the videos that are to be Upscaled, or videos to be de-interlaced before hand.

  • Interlaced Videos: "Interlaced_<EP ###>_<SHOWNAME>.<EXTENSION>"
  • Input Video: "Input_<EP ###>_<SHOWNAME>.<EXTENSION>"

/ImportAudio & /ImportSubtitles

These folders are where you put either all of your subtitles, or audio to be muxed together with the export video. The Stream Extraction Script will put files here on completion.

  • Audio Files: "<LANG ###>_<EP ###>_<SHOWNAME>.<EXTENSION>"
  • Subtitle Files: "<LANG ###>_<EP ###>_<SHOWNAME>.<EXTENSION>"

Rarely Accessed Folders

/QueuedFrames & /IntermediateFrames & /ProcessedFrames

These folders are used by FTU to store all of the frames it is manipulating. You will need to inspect these folders when using the Manual Upscale Script
Frames derived from Stage A, will go in /QueuedFrames
Frames created from Stage B, will go in either /IntermediateFrames if RawTherapee is used, or /ProcessedFrames if not
Frames created from Stage C, will go in /ProcessedFrames

  • Frames: "OEP######.<EXTENSION>"

/Queue

This folder is where all of the Upscale Processes are saved to disk. This folder should not be tampered with.

  • Process Serializations: "Q###.ser"

/Temp

This folder stores logs, and the preferences for FTU. FTU logs its own activities in foxtrot.log. Every time the program runs, FTU's log is rolled over. These log's are necessary when filing bug reports. Please only upload the logs in question, do not upload logs unrelated to any issues. foxtrot.log is the latest log, while foxtrot-1.log is the second latest, and so on. Another file will be GlobalData.ser, which stores all the preferences in FTU. Deleting this will reset FTU as if being freshly installed. The last file should be log.txt, which is a log used by Stage D.
None of these files should be deleted, only the logs should be shared with the developer, could hold sensitive information.

  • FTU logs: "foxtrot-#.log"
  • FTU Saved Data: "GlobalData.ser"
  • FFMPEG log: "log.log"

Individual Files

config.txt

This text file is so important and essential, it has its own section in the documentation, please refer to that.

rawTherapee.pp3

If you want to run Stage C, this file will be necessary, otherwise Stage C will be skipped. This file tells RawTherapee exactly what to do with each frame, before the frames are compressed together into a video. Must be generated by actually opening RawTherapee, and generating a .pp3 file in there.

running

Helps prevent running multiple instances of FTU from running at the same time. Only delete if FTU claims to be running already when it is not.
